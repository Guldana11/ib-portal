# ADMIN.md — Функционал администратора

## Роли и доступ

Роль `ADMIN` назначается вручную через БД или другим администратором.
Все маршруты `/api/admin/*` защищены middleware `isAuthenticated + isAdmin`.

---

## 1. Управление документами

### Загрузка нового документа

- Форма: название, категория, версия, описание, файл PDF (до 50 МБ)
- Файл сохраняется в MinIO с ключом `documents/{cuid}.pdf` (оригинальное имя — в поле `fileName`)
- После загрузки документ в статусе `isPublished: false` — сначала проверить, потом публиковать
- При публикации (`isPublished: true`) → отправить уведомление всем активным сотрудникам (см. раздел Email)

### Обновление версии документа

- Загрузка нового файла для существующего документа
- Поле `version` инкрементируется вручную администратором (например `1.0` → `1.1`)
- После сохранения все `DocumentAcknowledgment` с предыдущей версией теряют актуальность автоматически (логика в сервисе)
- → Уведомление сотрудникам: «Документ обновлён, требуется повторное ознакомление»

### Компонент DocumentUpload

```
[Название *]        [Категория ▼]
[Версия]            [Описание]
[Загрузить PDF  ← drag & drop или кнопка]
                    [Сохранить как черновик] [Опубликовать]
```

## 2. Управление тестами

### Редактор теста (QuestionEditor)

Привязан к конкретному документу. Один документ = один тест (создаётся отдельно, не автоматически).

Поля теста:
- Название теста
- Проходной балл (%, по умолчанию 80)
- Ограничение времени (минуты, необязательно)
- Максимум попыток (по умолчанию 3)
- Периодичность (дни, по умолчанию 365)

Редактор вопросов:
- Добавить/удалить вопрос
- Текст вопроса
- 2–6 вариантов ответа, отметить правильные (один или несколько)
- Пояснение (показывается сотруднику после сдачи)
- Drag & drop для изменения порядка вопросов

## 3. Управление пользователями

Таблица пользователей:
- Email, имя, роль, статус (активен/заблокирован), дата последнего входа
- Действия: сменить роль, заблокировать/разблокировать, сбросить попытки теста

**Сброс попыток** — отдельная кнопка рядом с пользователем, открывает модал: «Сбросить попытки по тесту [выбор теста]?»

## 4. Отчёты и дашборд

### Dashboard (сводка)

Карточки-метрики:
- Всего сотрудников / активных
- % ознакомившихся с каждым опубликованным документом
- % успешно сдавших тест
- Сотрудники с просроченным сроком переобучения (periodDays)

### ComplianceReport (детальный отчёт)

Таблица с фильтрами (по документу, по статусу, по дате):

| Сотрудник | Email | Документ | Версия | Ознакомлен | Тест сдан | Балл | Дата теста |
|-----------|-------|----------|--------|------------|-----------|------|------------|

Фильтры:
- Документ (мультивыбор)
- Статус ознакомления: все / ознакомлен / не ознакомлен / устарело
- Статус теста: все / сдан / не сдан / не проходил
- Период дат

### Экспорт

- Кнопка «Скачать CSV» — выгружает текущий отфильтрованный набор
- Кнопка «Отправить отчёт на email» — открывает модал с полем адреса (по умолчанию email текущего admin), отправляет CSV вложением

## 5. Email-уведомления

Используется **Nodemailer** + корпоративный SMTP (настройки в `.env`).
Все письма — на русском языке, plain text + HTML-версия.

### Шаблоны писем

#### 5.1 Новый документ опубликован
**Кому:** все активные сотрудники (`role: EMPLOYEE, isActive: true`)
**Триггер:** администратор нажимает «Опубликовать»

```
Тема: [Crystal Spring ИБ] Новый документ для ознакомления: {document.title}

Уважаемый(ая) {user.name},

Опубликован новый документ по информационной безопасности:
«{document.title}» (версия {document.version})

Пожалуйста, ознакомьтесь с документом в течение 5 рабочих дней:
{FRONTEND_URL}/documents/{document.id}

С уважением,
Служба информационной безопасности ТОО «Crystal Spring»
```

#### 5.2 Документ обновлён
**Кому:** сотрудники, у которых есть устаревший `DocumentAcknowledgment`
**Триггер:** публикация новой версии документа

```
Тема: [Crystal Spring ИБ] Документ обновлён, требуется повторное ознакомление

Уважаемый(ая) {user.name},

Документ «{document.title}» обновлён до версии {document.version}.
Ваше предыдущее ознакомление (версия {oldVersion}) больше не актуально.

Ознакомьтесь с обновлённой версией:
{FRONTEND_URL}/documents/{document.id}
```

#### 5.3 Напоминание о прохождении теста
**Кому:** сотрудники, у которых истёк `Test.periodDays` с момента последней успешной сдачи (или никогда не сдавали)
**Триггер:** ручная рассылка администратором ИЛИ cron (еженедельно по понедельникам)

```
Тема: [Crystal Spring ИБ] Напоминание: требуется сдать тест по ИБ

Уважаемый(ая) {user.name},

Напоминаем о необходимости пройти тест:
«{test.title}» (к документу «{document.title}»)

{если never}: Вы ещё не проходили данный тест.
{если expired}: Последняя успешная сдача: {date}. Срок действия истёк.

Пройти тест: {FRONTEND_URL}/tests/{test.id}
```

#### 5.4 Отчёт по соответствию (вложение CSV)
**Кому:** на email, указанный администратором
**Триггер:** кнопка «Отправить отчёт на email» в разделе отчётов

```
Тема: [Crystal Spring ИБ] Отчёт по соответствию — {дата}

Во вложении CSV-отчёт по состоянию ознакомления сотрудников
с документами и результатам тестирования на {дата}.

Фильтры применённые при выгрузке: {фильтры или "без фильтров"}
```

### Очередь отправки

При массовой рассылке (все сотрудники) — отправлять через очередь, **по одному письму**, с задержкой 100–200 мс между письмами, чтобы не попасть в спам и не перегрузить SMTP. Использовать простой in-memory цикл с `setTimeout` (без BullMQ/Redis для простоты).

```typescript
// notificationService.ts
async function sendBulk(users: User[], templateFn: (u: User) => MailOptions) {
  for (const user of users) {
    await transporter.sendMail(templateFn(user));
    await new Promise(r => setTimeout(r, 150));
  }
}
```

## 6. Ручные действия администратора (UI)

| Действие | Где | Результат |
|----------|-----|-----------|
| Опубликовать документ | DocumentUpload | Email всем сотрудникам |
| Обновить версию | DocumentUpload | Email тем, кто уже ознакомился |
| Разослать напоминание о тесте | ComplianceReport → кнопка «Напомнить» | Email выбранным / всем |
| Отправить отчёт на email | ComplianceReport → кнопка | CSV вложением |
| Сбросить попытки теста | UserTable → кнопка | Сотрудник может пройти снова |
